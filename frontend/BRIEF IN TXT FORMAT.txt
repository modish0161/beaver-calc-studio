BeaverCalc Studio — Comprehensive Product Brief
(Structural Engineering Calculations Platform)
Prepared for Beaver Bridges Ltd
Date: 11 November 2025
———————————————————————————————————
1. Executive Summary
BeaverCalc Studio is a modern, modular structural and temporary-works calculations platform designed for bridge projects and allied civil engineering tasks. It combines a Python calculations engine with a Flask API bridge and a React + TypeScript frontend. The system focuses on rigorous engineering methods (Eurocodes and UK practice), robust auditability, versioned calculations, and rapid, elegant reporting for design and site teams.
2. Objectives & Non‑Objectives
2.1 Objectives
•	Provide a single hub for structural and temporary-works calculations used by Beaver Bridges engineers.
•	Deliver repeatable, version‑controlled results with transparent inputs, assumptions, and code references.
•	Enable fast report creation (PDF/DOCX) with project branding and drawing references.
•	Offer a plugin architecture so new calculators can be added safely without impacting the core.
•	Support UK practice (Eurocodes with UK NAs; BS 5975 for temporary works; common BRE/industry guidance).
•	Integrate with existing tools and file formats (DWG/DXF exports, IFC where appropriate).
2.2 Non‑Objectives (Initial Release)
•	Full 3D FEA for arbitrary geometry (roadmap item; initial release covers beam/frame/grillage solvers).
•	Automated BIM round‑tripping to Revit (initially export‑first via DXF/IFC).
•	Site mobile app (mobile‑web responsive UI is in scope; native apps are a later phase).
3. Users & Personas
•	Structural Engineer (bridges): designs decks, beams, bearings, connections, and checks load effects.
•	Temporary Works Coordinator (TWC/TWS): designs crane pads, working platforms, falsework, and access.
•	Site Engineer / Supervisor: needs quick checks, reports, and method‑statement extracts.
•	Senior Project Manager: reviews outputs, signs off assumptions, tracks versions and approvals.

5. System Architecture
Architecture motif: Python calculations engine ⟶ Flask API bridge ⟶ React + TypeScript frontend.
•	Backend (Python): calculations engine (NumPy/SciPy), units via Pint, validation via Pydantic, persistence via SQLAlchemy + PostgreSQL, reporting via Jinja2→HTML→PDF/DOCX.
•	API Layer (Flask): REST endpoints, OpenAPI generation, JWT-based auth, role‑based access control (RBAC), Flask‑SocketIO for live job progress and streaming logs.
•	Frontend (React + TypeScript): Vite, shadcn/ui + Tailwind, TanStack Query for data, react‑hook‑form + Zod for schema‑driven forms, Recharts for plots.
•	Task Execution: Redis + RQ (or Celery) for long‑running calculations with resumable jobs.
•	Files/Exports: Local or S3‑compatible object storage for reports, spreadsheets, and DXFs.
•	Observability: Structured logging, request IDs, Sentry (optional) for error tracking.

7. API Design (Flask)
Selected endpoints (non‑exhaustive):
•	POST /auth/login — obtain JWT
•	GET /calculators — list calculators and schemas
•	GET /calculators/{key} — get calculator metadata (inputs/outputs, units, references)
•	POST /runs — start a calculation run with input payload; returns run_id
•	GET /runs/{run_id} — fetch status/results
•	WS /runs/{run_id}/stream — live progress/logs via WebSocket
•	POST /reports — render DOCX/PDF for a completed run
•	GET /materials/steel, /materials/concrete, /sections/ukb — libraries and catalogues
•	GET /projects/{id}/files — retrieve artefacts; PUT for uploads (drawings, photos)

9. Calculator Framework (Plugin Model)
•	Each calculator ships as a Python package module with:
•	• Schema.py — Pydantic models for inputs/outputs with units
•	• Engine.py — deterministic functions using NumPy/SciPy
•	• Checks.py — limit‑state checks and utilisation aggregation
•	• Report.md.j2 — Jinja template for human‑readable output
•	• Tests/ — unit tests with golden files for verification
Example: Steel Beam Bending Check — Skeleton
class Inputs(BaseModel):
    section: str
    span_m: float
    uniform_load_kN_per_m: float
    lateral_restraint: str
    steel_grade: str

class Outputs(BaseModel):
    M_Ed_kNm: float
    V_Ed_kN: float
    utilisation_bending: float
    utilisation_shear: float
    deflection_mm: float

10. Reporting & Exports
•	DOCX/PDF reports with Beaver Bridges branding and drawing references.
•	Appendices with assumptions, formulae, code clauses, and intermediate steps (optional).
•	DXF overlays for pad footprints, load patches, and influence areas (export).

11. Security, Compliance & Audit
•	Role‑based access: Admin, Designer, Checker, Viewer.
•	Full audit trail for inputs, results, and report issuance.
•	Immutable run hashes so any change invalidates signatures.
•	GDPR‑aligned data handling; on‑prem or VPC deployment.
•	Disclaimer blocks and sign‑off workflows for issued reports.
12. UX Principles
•	Clear, friendly forms with unit pickers and inline validation.
•	‘Explain this’ toggles reveal methods, clauses, and equations.
•	Save as template: reuse standard assumptions and materials.
•	Dark/light themes; keyboard navigation; responsive layouts.
13. Developer Experience & CI/CD
•	Monorepo with /backend (Flask) and /frontend (React‑TS).
•	Dev containers (VS Code) with pinned toolchains.
•	GitHub Actions: lint, type‑check, tests, build images, publish artifacts.
•	Smoke tests run against a docker‑compose environment.
14. Deployment Options
•	On‑prem Windows Server or Linux VM via Docker Compose.
•	Azure (Web App + Postgres Flexible Server + Redis) with private networking.
•	Backups and retention policies for DB and object storage.

16. Risks & Mitigations
•	Scope creep → Strict P0/P1/P2 gating; fortnightly demos.
•	Validation debt → Golden tests; dual‑calc spot checks against hand calcs and trusted spreadsheets.
•	Performance on large load cases → Async jobs + caching; optimise hotspots in NumPy.
•	Change management → Version pinning and migration scripts; clear release notes.
